# Naver Separated JPG Workflow

Use this reference whenever `$francise-detail-page-naver` is asked to create a Naver blog-connected franchise detail page.

## Core Output Contract

- Default output is not HTML and not one long scrolling image.
- Create 12 separate JPG files unless the user asks for another exact count.
- Each JPG must be `1080x1080`, named `cut-01.jpg` through `cut-12.jpg`.
- Also create `blog-caption-copy.md` with the explanatory Korean blog text to place between images.
- Keep image text short. Images are visual cards; the blog body carries the detailed explanation.
- Use real photos from the prepared franchise photo library when available.

## Why This Matters

The user's outsourced blog examples use separate square cards, usually 11 images plus one text file. Those cards work as visual anchors inside a Naver blog post, not as a full article trapped inside images. For the user's current workflow, use that structure and add one more card when a 12-cut detail-page flow is requested.

## Visual Pattern From Existing Outsourced Samples

- Square card size: `1080x1080`.
- Set style: separate cards inserted between blog paragraphs.
- Typical flow:
  - First card: region, brand, startup/takeover topic.
  - Early card: condition table or key numbers.
  - Middle cards: real store photos with short title overlays.
  - Analysis cards: darkened real-photo background with 2-4 concise points.
  - Last card: consultation CTA and disclaimer.
- Color system:
  - Dark brown or deep neutral background.
  - Brand accent color used sparingly.
  - White bold Korean text.
  - Simple rectangular/table blocks, not dense editorial pages.
- Avoid overfilling the image. If a sentence needs explanation, move it to `blog-caption-copy.md`.

## 12-Cut Naver Franchise Flow

1. Title card: region + brand + core decision question.
2. Condition summary: monthly sales, startup cost, location, BI/store condition.
3. Exterior/signage card: visibility and access.
4. New startup vs takeover reason: why existing data matters.
5. Interior/BI card: store condition and first impression.
6. Sales check card: provided sales number as verification point, not guarantee.
7. Trade-area photo card: station, road, surrounding flow.
8. Trade-area analysis card: demand, competition, time-zone flow.
9. Operation card: counter, menu, pickup, staff flow.
10. Startup cost review: included assets, facility status, extra cost risk.
11. Pre-contract checklist: sales data, headquarters transfer approval, facilities, contract, included scope.
12. CTA: practical conclusion, analysis offer, and brand-safety disclaimer.

## Blog Caption Copy

For each cut, write a matching explanation block:

- Use a descriptive subheading.
- Explain what the image alone cannot explain.
- Keep one section focused on one buyer question.
- Mention provided facts carefully.
- Do not use exaggerated promise language.
- Use consulting language such as `검토`, `확인`, `비교`, `분석`, `확인 필요`.

## Marketing QA

- Does the first card answer why this opportunity deserves attention?
- Does the first blog paragraph contain the practical conclusion before long context?
- Are the numbers connected to verification logic, not used as bait?
- Is there a clear reason this region/store differs from a generic brand post?
- Does the copy help first-time buyers reduce uncertainty?
- Are risks and checks included before the CTA?

## Design QA

- Are all images `1080x1080` JPGs?
- Can each card stand alone when viewed in the Naver mobile feed?
- Is the Korean readable on a phone?
- Is the image copy short enough that the blog body still has a job?
- Are photo cards and text cards visually consistent?
- Does the page avoid looking like official headquarters material?
- Are faces, phone numbers, and unrelated private details avoided or cropped?

## Compliance QA

- Use `창업비용`.
- Do not use `권리금`.
- Do not expose rent or deposit unless the user explicitly asks.
- Do not say sales, profit, success, or payback is guaranteed.
- Do not imply headquarters approval unless verified.
- Include: `본 콘텐츠는 양도양수 상담/분석용이며 본사 공식 자료가 아닙니다.`
